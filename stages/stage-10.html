💬 کامنت‌گذاری در پایتون

یادداشت‌گذاری روی کدها برای خودت و دیگران

👋 سلام، دانشجو جان! امروز یاد می‌گیریم چطور کدهایمان را خواناتر کنیم 💬

💬 کامنت (Comment) چیست؟

کامنت یعنی یادداشت یا توضیح که داخل کد می‌نویسی، اما پایتون آن را اجرا نمی‌کند و کاملاً نادیده می‌گیرد! کامنت‌ها فقط برای انسان‌ها نوشته می‌شوند، نه برای کامپیوتر.

تصور کن یه دفترچه یادداشت کنار دستته و هر جا لازم بود، یه توضیح کوچک می‌نویسی تا بعداً خودت یا دیگران بفهمند که این قسمت از کد چه کاری انجام می‌دهد. کامنت دقیقاً همین کار را می‌کند!

🎯 تعریف ساده: کامنت = متن‌هایی در کد که فقط برای خواندن انسان است و پایتون آنها را کاملاً نادیده می‌گیرد. کامنت‌ها روی اجرای برنامه هیچ تأثیری ندارند!

✍️ انواع کامنت در پایتون

۱. کامنت تک‌خطی (Single-line Comment)

با علامت # (هشتگ/شارپ) شروع می‌شود. هر چیزی که بعد از # در آن خط بنویسی، کامنت محسوب می‌شود و اجرا نمی‌شود.

# این یک کامنت است - پایتون این خط را نادیده می‌گیرد print("Hello World!") # این هم کامنت است # کامنت می‌تواند توضیح دهد که کد چه کاری انجام می‌دهد name = "Ali" # ذخیره اسم کاربر age = 25 # ذخیره سن کاربر

۲. کامنت چندخطی (Multi-line Comment)

با سه علامت کوتیشن (""" """ یا ''' ''') ساخته می‌شود. هر چیزی بین این سه‌تایی‌ها بنویسی، کامنت محسوب می‌شود. این روش برای توضیحات طولانی عالی است!

""" این یک کامنت چند خطی است. می‌توانی اینجا چندین خط توضیح بنویسی. پایتون همه این خطوط را نادیده می‌گیرد. خیلی عالی برای توضیح تابع‌ها و کلاس‌ها! """ print("این کد اجرا می‌شود") ''' این هم یک کامنت چند خطی دیگر با سینگل کوتیشن سه‌تایی. کاملاً معادل روش قبلی است. '''

🎯 چرا کامنت‌گذاری مهم است؟

شاید الان فکر کنی: «خب که چی؟ من خودم کد رو نوشتم، می‌دونم چیکار می‌کنه!» اما باور کن چند هفته بعد که به کدت نگاه کنی، ممکنه هیچی یادت نیاد! 😅

📌 دلایل مهم کامنت‌گذاری:
  • 🧠 یادآوری به خودت: وقتی بعداً به کد برمی‌گردی، می‌فهمی چه کاری انجام می‌دهد
  • 👥 کمک به دیگران: اگر کدت را با کسی به اشتراک بگذاری، او هم می‌فهمد
  • 🐞 دیباگ کردن: می‌توانی موقتاً یک خط کد را کامنت کنی تا ببینی بدون آن برنامه چطور کار می‌کند
  • 📚 مستندسازی: کامنت‌ها مثل دفترچه راهنمای کد تو عمل می‌کنند

🔧 استفاده از کامنت برای دیباگ (Debugging)

یکی از کاربردهای جالب کامنت، غیرفعال کردن موقت بخشی از کد برای پیدا کردن خطاهاست. به این کار می‌گویند Comment Out.

print("خط ۱: این اجرا می‌شود") # print("خط ۲: این را موقتاً غیرفعال کردم") print("خط ۳: این هم اجرا می‌شود")

اینطوری می‌تونی ببینی اگر خط ۲ نباشد، برنامه چطور کار می‌کند، بدون اینکه واقعاً آن خط را پاک کنی!

📝 اصول و قوانین کامنت‌گذاری حرفه‌ای

کامنت‌گذاری هم آدابی دارد! یک برنامه‌نویس حرفه‌ای کامنت‌های مفید و مختصر می‌نویسد، نه کامنت‌های بدیهی و بی‌فایده!

❌ کامنت بد (بدیهی و بی‌فایده):
x = 5 # عدد 5 را در x قرار بده
این کامنت هیچ اطلاعات جدیدی نمی‌دهد! از کد پیداست که 5 در x قرار می‌گیرد.
✅ کامنت خوب (توضیح هدف و دلیل):
x = 5 # حداقل نمره قبولی در درس ریاضی
این کامنت می‌گوید چرا عدد 5 انتخاب شده، نه اینکه چه کاری انجام می‌شود.

📋 قوانین طلایی کامنت‌گذاری:

  • ✅ هدف را توضیح بده، نه عمل را: بگو «چرا» این کد نوشته شده، نه «چه کاری» می‌کند
  • ✅ مختصر و مفید باش: کامنت نباید رمان باشد! یک یا دو خط کافی است
  • ✅ کد پیچیده را توضیح بده: اگر الگوریتم یا منطق پیچیده‌ای داری، حتماً توضیح بده
  • ✅ به‌روز نگه دار: اگر کد را تغییر دادی، کامنتش را هم تغییر بده
  • ❌ کامنت بی‌فایده ننویس: کامنتی که فقط کد را تکرار کند، ارزش ندارد
  • ❌ کامنت را جایگزین کد خوب نکن: کد تمیز و خوانا بهتر از کامنت طولانی است

🌍 کامنت‌گذاری به فارسی یا انگلیسی؟

پایتون از کاراکترهای فارسی (یونیکد) در کامنت‌ها پشتیبانی می‌کند. پس می‌توانی کامنت‌هایت را به فارسی بنویسی! اما اگر در پروژه‌های بین‌المللی کار می‌کنی، انگلیسی بهتر است.

# این کامنت فارسی است و پایتون آن را قبول دارد! # This is an English comment print("Both are fine!")

⌨️ کلیدهای میانبر برای کامنت‌گذاری در VS Code

در VS Code می‌توانی با یک کلید میانبر، یک یا چند خط را سریعاً کامنت کنی:

  • 🪟 ویندوز: Ctrl + /
  • 🍎 مک: Cmd + /

خط(های) انتخاب شده را هایلایت کن، این کلیدها را بزن، و به‌طور خودکار # به ابتدای خط اضافه می‌شود. دوباره بزنی، # حذف می‌شود!

🧪 تمرین‌های عملی کامنت‌گذاری

روی هر کارت کلیک کن تا پاسخ را ببینی! اول خودت حل کن، بعد جواب را چک کن!

۱
یک کامنت تک‌خطی به کد زیر اضافه کن که توضیح دهد چه کاری می‌کند
👆 کلیک کن تا جواب را ببینی
✅ پاسخ:
# دریافت نام کاربر و خوش‌آمدگویی name = input("اسمت چیه؟ ") print("سلام", name)
۲
خط دوم را موقتاً کامنت کن تا اجرا نشود
👆 کلیک کن تا جواب را ببینی
✅ پاسخ:
print("خط یک") # print("خط دو") print("خط سه")
۳
یک کامنت چندخطی برای توضیح برنامه بنویس
👆 کلیک کن تا جواب را ببینی
✅ پاسخ:
""" برنامه محاسبه مساحت مستطیل نویسنده: علی تاریخ: ۱۴۰۲ """ length = 10 width = 5 area = length * width print("مساحت:", area)
۴
کدام کامنت بهتر است؟ چرا؟
👆 کلیک کن تا جواب را ببینی
✅ پاسخ:
# ❌ بد: x = x + 1 # یکی به x اضافه کن # ✅ خوب: x = x + 1 # شمارنده دانش‌آموزان # دلیل: دومی هدف را توضیح می‌دهد
۵
با کلید میانبر، چطور یک خط را سریع کامنت می‌کنی؟
👆 کلیک کن تا جواب را ببینی
✅ پاسخ:
ویندوز: Ctrl + / مک: Cmd + / خط را انتخاب کن، کلیدها را بزن!
۶
یک برنامه ساده با input و print بنویس و با کامنت توضیح بده
👆 کلیک کن تا جواب را ببینی
✅ پاسخ:
# برنامه خوش‌آمدگویی به کاربر name = input("اسمت چیه؟ ") # نمایش پیام خوش‌آمد print(f"خوش آمدی {name} جان!")
